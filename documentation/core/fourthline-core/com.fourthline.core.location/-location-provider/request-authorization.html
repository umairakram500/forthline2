requestAuthorization

abstract fun requestAuthorization(success: (status: LocationProvider.AuthorizationStatus) -> Unit, failure: (error: LocationProviderError) -> Unit)
Content copied to clipboard

Request location permission. If authorization already requested and has status AuthorizationStatus.AUTHORIZED or AuthorizationStatus.DENIED, method returns this status immediately in the success callback. If authorization is in AuthorizationStatus.NOT_DETERMINED state, show native Android popup to the user to request location permissions and return AuthorizationStatus.AUTHORIZED in success callback if user granted permission and AuthorizationStatus.DENIED is user denied it.

Parameters

success

callback that provides AuthorizationStatus.

failure

callback that provides LocationProviderError.

Request will fail with LocationProviderError.AnotherRequestInProgress if another LocationProvider request is in progress. Request will fail with LocationProviderError.LocationServicesNotEnabled if location services not enabled at the device's settings. Request will fail with LocationProviderError.Unknown if unexpected exception occur during request.
