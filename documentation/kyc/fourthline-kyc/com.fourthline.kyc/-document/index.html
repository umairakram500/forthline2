Holds document information.

Parameters

type

required, value should be set.

number

required, value should NOT be null, empty, blank or has character count outside of [6-16] range, e.g. "SPECI 2014".

issueDate

optional, when set, date should NOT be in future.

expirationDate

optional, when set, date is not validated.

images

required, list should NOT be null or empty and should contain properly arranged Attachment.Documents. There is a special validation rule for lists that contain angled images (see Attachment.Document.isAngled). Every such image should be preceded with a normal non-angled image of the same Attachment.Document.fileSide. For example:

Valid:

{[fileSide = FRONT, isAngled = false]};

{[fileSide = FRONT, isAngled = false], [fileSide = BACK, isAngled = false], ...};

{[fileSide = FRONT, isAngled = true], [fileSide = FRONT, isAngled = false], ...}.

Invalid:

{[fileSide = FRONT, isAngled = false], [fileSide = FRONT, isAngled = true], ...};

{[fileSide = FRONT, isAngled = true], [fileSide = BACK, isAngled = false], ...}.

nfc

optional, when set, value should be valid (Attachment.Nfc.validate should not return any errors).

videoRecording

optional, when set, the video should have a duration of less than 10 seconds when VideoRecording.duration is VideoDuration.DEFAULT or less than 60 seconds when VideoRecording.duration is VideoDuration.EXTENDED.
